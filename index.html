Git and Github short Tutorial

Command line basics

check version

$ git --version

Displays Git version.

check config settings

$ git config --list

Displays my Git configurations.

git help

$ git help
$ git help <verb>
$ git <verb> --help

Displays either total list of help or a help for the quotation keyword.

Initialize a repo from existing code

$ git init

Initializes a local repo (creates  .git hidden file).


$ rm -rf .git

Removes  .git directory (like any other directory)

Before first commit

$ git status

Shows current status - which files are tracked, untracked, modified ets.


$ touch .gitignore

Creates a  .gitignore file where we can keep files which we woudn't want to track.


Add files to staging aria

$ git add <file name>
$ git add -A
$ git add .

Adds either certain file or all files to the staging area.

Remove files from staging area

$ git reset <file name>

Removes certain file from the staging area.


$ git reset

Removes all files from the staging area.

Commit

$ git commit -m "here description of the commit"

Commits files from the staging area to local repo.


$ git commit -am "here description of the commit"

Adds files to the staging and commits them simultaneously area to local repo.


$ git log

Provides data about commits.

Cloning a remote repo

$ git clone <url> <where to clone>
$ git clone <url> .

Clones a remote repo to a local mashine (to current directory).

View info about remote repo

$ git remote -v

Provides info about remote repo.

Push commits to remote repo

$ git pull origin master

Before pushing need to pull updated data from the remote repo.


$ git push origin master

Push changes I've made to the remote repo.

Common workflow

Create a branch for desired feature

$ git branch branch-name

Creates a new branch on local mashine.


$ git checkout -b -name

Creates a new branch on local mashine and switches to it at once.


$ git branch

Lists all local branches.


$ git checkout branch-name

Switches to the named branch.


After commit push branch to remote

$ git push -u origin branch-name

Pushes new branch to the remote repo.


$ git branch

Lists all local branches.


$ git checkout branch-name

Switches to the named branch.


$ git branch -a

Lists all branches, local and remote.


Merge a branch

$ git checkout master

Switch to the local master branch.


$ git diff branch-name

Shows changes one can have made to the code.


$ git pull origin master

Pull latest master branch from the remote repo, in case of dealing with the remote.


$ git branch --merged

Shows if any merges were done.


$ git merge branch-name

Merges the feature branch to the master branch.


$ git push origin master

Pushes updated, merged master branch to the remote.


Deleting a branch

$ git branch --merged

Double check all was done correctly.


$ git branch -d branch-name

Deletes named branch locally.


$ git branch -a

Shows that the feature branch was deleted locally, but it still exists on the remote repo.


$ git push origin --delete branch-name

Pushes updated data and deletes named branch on remote repo.


$ git push origin master

Pushes updated, merged master branch to the remote.
