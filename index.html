Git and Github short Tutorial

Command line basics

General info

$ git --version

Displays Git version.

$ git config --list

Displays Git configurations.

$ git init

Initializes a local repo (creates  .git hidden file).

$ rm -rf .git

Removes  .git directory (like any other directory)

$ git help
$ git help <verb>
$ git <verb> --help

Displays either total list of help or a help for the quotation keyword.

$ git status

Shows current status - which files are tracked, untracked, modified ets.

$ touch .gitignore

Creates a  .gitignore file, which contains untracked folders and files.

$ ls -1 -a

or

$ ls -1a

Lists all files. -1:  in one column; -a:  including hidden.

$ cat <file name>

Prints file's content.

$ git diff

Prints changes in a file.

$ git diff --staged

Prints changes in a staged file.

$ git diff branch-name

Shows changes between branches.

$ git log --oneline -all

Prints detailed data about commits. --oneline:  in one line for each commit;   -all:  including all commits, if requested from somewhere in the middle of the tree.

$ git mv file-name.ext /dir-name new-file-name.ext

Moves file-name.ext to another directory and changes its name to new-file-name.ext, simultaneously adds it to index.

Add/Remove files to/from the staging aria

$ git add <file name>
$ git add -A
$ git add .

Adds either certain file or all files to the staging area.

$ git reset <file name>

Removes certain file from the staging area.

$ git reset

Removes all files from the staging area.

Commit

$ git commit -m "Description of the commit."

Commits files from the staging area to the local repo.

$ git commit -am "Description of the commit."

Adds files to the staging and commits them simultaneously to the local repo.

$ git rm <dir/file-name>

Removes <file-name> from commit AND from working directory.

$ git rm --cached <dir/file-name>

Removes <file-name> from commit ONLY.

$ git commit --amend -m 'Corrected text here.'

Possible to make corrections to the LAST commit description.

$ git commit --amend --no-edit

Makes a commit with the previous description.

$ git restore
$ git restore <file name>

Undo changes and get back to the last commit (the whole project or specified file).

$ git restore --staged <file name>

Undo changes and withdraw from index (the whole project or specified file).

Dealing with remote repo

Cloning a remote repo

$ git clone <url> <where to clone>
$ git clone <url> .

Clones a remote repo to a local mashine (to current directory).

View info about remote repo

$ git remote -v

Provides info about remote repo.

Push commits to remote repo

$ git pull origin master

Before pushing need to pull updated data from the remote repo.

$ git push -u origin branch-name

Pushes new branch to the remote repo.

$ git push

Pushes changes to the remote repo.

Common workflow

Branches

$ git branch

Lists all local branches and marks current branch.

$ git branch -M main

Will rename current branch to <main>.

$ git branch -a

Shows branches on the remote repo.

$ git branch branch-name

Creates a new branch on a local mashine.

$ git checkout -b -name

Creates a new branch on a local mashine and switches to it at once.

$ git checkout branch-name

Switches to the named branch.

$ git branch -d/-D branch-name

Deletes named branch locally.

$ git push origin --delete branch-name

Pushes updated data and deletes named branch on remote repo.

Move around commits

$ git show ec7489d

Shows changes in a certain commit.

$ git cat-file -p ec7489d

Shows info of the specified commit (ec7489d). -p will print info readeble for human.

$ git checkout ec7489d

Will move to the specified commit (ec7489d).

$ git checkout -b commit-name ec7489d

Will move to the specified commit (ec7489d) and will attach a commit-name to it.

Merge a branch

$ git checkout master

Switch to the local master branch.

$ git pull origin master

Pull latest master branch from the remote repo, in case of dealing with the remote.

$ git branch --merged

Shows if any merges were done.

$ git merge branch-name

Merges the feature branch to the master branch.

$ git push origin master

Pushes updated, merged master branch to the remote.
